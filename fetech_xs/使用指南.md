# ftech_xs 使用指南

## 一、简介

ftech_xs 是一个小说下载与 EPUB 生成工具，专门用于从爱丽丝书屋 (ALICESW.COM) 下载小说章节并合成为 EPUB 电子书。

**核心功能：**
- 自动解析章节列表页面，提取所有章节链接
- 逐章下载章节内容（HTML 格式，保留段落结构）
- 内置反爬应对策略（延迟15~30秒、Cookie维持、空内容自动重试+API切换）
- 支持断点续传（中断后重新运行会自动继续）
- 支持 `--rd` 重试失败章节
- 下载完成后一键生成带章节号的 EPUB 目录

## 二、环境准备

### 2.1 安装 Python

确保系统已安装 Python 3.13+。验证方式：

```powershell
py -3 --version
```

如果显示 `Python 3.13.5` 或类似版本，说明环境就绪。
如果提示找不到 `py`，请先安装 Python：https://www.python.org/downloads/

### 2.2 安装依赖

在 `fetech_xs/` 目录下运行：

```powershell
py -3 -m pip install -r requirements.txt
```

该命令会自动安装以下依赖：
| 包名 | 用途 |
|------|------|
| requests | HTTP请求与Session管理 |
| beautifulsoup4 | HTML解析 |
| lxml | 解析引擎加速 |
| ebooklib | EPUB电子书生成 |

## 三、快速开始

### 3.1 下载小说

```powershell
# 进入脚本目录
cd fetech_xs

# 方式一：用 book_id 下载（推荐）
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py 32157

# 方式二：用完整URL下载（适用于其他小说）
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py "https://xn--1jqx9a.als1004.space/other/chapters/id/32157.html"
```

**book_id 如何获取？**
打开你想下载的小说的"章节目录"页面，URL 末尾的数字就是 book_id。例如：
```
.../other/chapters/id/32157.html   ← 32157 就是 book_id
```

### 3.2 生成 EPUB

下载完成后，运行：

```powershell
$env:PYTHONIOENCODING='utf-8'; py -3 build_epub.py 召唤圣剑
```

其中"召唤圣剑"是脚本自动创建的小说目录名（与小说网页上的书名一致）。
生成的文件在 `fetech_xs/召唤圣剑/召唤圣剑.epub`。

生成的 EPUB 目录中会显示完整的章节名称，例如：
```
第一章 新的旅程
第二章 圣剑套牌
第三章 遇袭
```

## 四、重试失败章节（--rd 选项）

如果在下载过程中某些章节因为网络原因失败，它们会被记录在 `progress.json` 的 `failed` 列表中。下载状态变为 `complete` 后，可通过 `--rd` 选项重新下载这些失败章节：

```powershell
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py --rd 32157
```

**运行逻辑：**
1. 读取 `progress.json` 中的 `failed` 列表
2. 从 `failed` 中移除待重试章节，立即保存状态（防止中途崩溃丢失记录）
3. 仅下载这些失败章节
4. 下载成功的章节加入 `downloaded`，失败的重新加入 `failed`
5. 如果所有失败章节都重试成功，状态恢复为 `complete`

**查看当前是否有失败章节：** 直接运行 `download_novel.py 32157`，如果状态为 `complete`，会提示"如需重试失败章节，请添加 --rd 参数运行"。

## 五、检查实际文件状态（--check 选项）

如果 `progress.json` 中的记录与 `chapters/` 目录里的实际文件不一致（例如手动删除了某些文件、脚本崩溃导致状态错乱），可以使用 `--check` 选项修复：

```powershell
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py --check 32157
```

**运行逻辑：**
1. 从网页抓取所有章节标题，预填到 `chapter_titles`
2. 扫描 `chapters/` 目录，检查哪些章节文件实际存在
3. 将 `downloaded` 和 `failed` 列表重置为实际文件状态
4. 缺失章节自动加入 `failed` 列表，便于后续用 `--rd` 重试

**典型场景：**
- 之前下载因崩溃中断，`failed` 列表不准 → 用 `--check` 修复后再 `--rd`
- 手动删除某些章节文件后需要重新下载 → 用 `--check` 自动发现缺失
- 怀疑 `progress.json` 状态有误 → 用 `--check` 一键校正

## 六、下载流程详解

### 6.1 第一步：解析章节列表

脚本访问章节列表页面，自动提取：
- **书名**：从 `<h1>《书名》</h1>` 中提取
- **作者**：从 `作者：<a>` 中提取
- **章节列表**：从 `<ul class="mulu_list">` 中提取所有含"章"的链接

### 6.2 第二步：初始化

```
1. 从网页抓取所有章节标题，预填到 progress.json 的 chapter_titles
   （确保 chapter_titles 始终完整准确，不再逐章追加）
2. 加载断点续传进度，跳过已下载章节
```

### 6.3 第三步：逐章下载

```
每章循环：
  1. 随机等待 15~30 秒（模拟人工阅读，减轻服务器压力）
  2. 优先访问 HTML 页面下载内容
  3. 如果HTML返回空内容 → 自动切换到 AJAX API 接口重试
  4. 如果仍失败 → 按指数退避等待（30s, 60s, 120s, 240s, 480s）后重试
  5. 成功后保存为 <序号>_<标题>.html
  6. 立即更新 progress.json（断点续传依据）
```

### 6.4 第四步：生成 EPUB

脚本读取 `chapters/` 目录下的所有 HTML 文件，按序号排序后：
1. 从 `progress.json` 的 `chapter_titles` 读取完整标题（含"第X章"）
2. 创建 EpubBook 对象，设置书名、作者、语言
3. 为每章生成 EpubHtml（含章节标题样式 + 内容）
4. 自动生成目录（TOC），显示完整章节名，兼容 EPUB2 和 EPUB3 阅读器
5. 输出 `.epub` 文件

## 七、断点续传

任何时候中断下载（按 Ctrl+C 或关机），重新运行相同的命令即可自动继续：

```powershell
# 重新运行，会从上次中断处继续
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py 32157
```

**原理：** 每成功下载一章后，都写入 `progress.json` 记录已下载的章节索引。下次启动时跳过这些章节。

## 八、下载多本小说

多次运行脚本即可，每本小说的章节会独立存放在各自的书名目录下：

```
fetech_xs/
├── 召唤圣剑/         ← 第一本书
│   ├── chapters/
│   └── progress.json
├── 另一本小说/       ← 第二本书
│   ├── chapters/
│   └── progress.json
└── download_novel.py
```

```powershell
py -3 download_novel.py 32157                         # 下载《召唤圣剑》
py -3 download_novel.py <另一本书的ID>                # 下载另一本书
py -3 build_epub.py 召唤圣剑                          # 生成第一本的EPUB
py -3 build_epub.py 另一本小说                        # 生成第二本的EPUB
```

## 九、配置文件说明

`config.json` 位于 `fetech_xs/` 根目录，可以通过修改它来调整脚本行为：

```json
{
    "request_delay": {
        "min": 15,    // 每章最小等待时间（秒）
        "max": 30     // 每章最大等待时间（秒）
    },
    "retry": {
        "max_attempts": 5,       // 单章最大重试次数
        "backoff_base": 30,       // 首次重试等待时间（秒）
        "backoff_multiplier": 2   // 每次重试乘数（30→60→120→240→480）
    },
    "headers": {
        "User-Agent": "...",      // 浏览器标识
        "Accept": "...",          // HTTP请求头
        "Accept-Language": "zh-CN,zh;q=0.9",
        "Accept-Encoding": "gzip, deflate",
        "Connection": "keep-alive",
        "Cache-Control": "no-cache"
    }
}
```

**重要提醒：** 请合理设置 `request_delay` 的值，不要设置过小，避免给服务器造成过大负担。默认的15~30秒是安全合理的间隔。

## 十、进度文件说明（progress.json）

```json
{
    "book_name": "召唤圣剑",
    "author": "西贝猫",
    "chapters_url": "https://...",
    "total": 679,
    "downloaded": [0, 1, 2],
    "failed": [5, 12],
    "last_index": 2,
    "status": "in_progress",
    "chapter_titles": {
        "0": "第一章 新的旅程",
        "1": "第二章 圣剑套牌",
        "2": "第三章 遇袭"
    }
}
```

| 字段 | 说明 |
|------|------|
| `chapter_titles` | 完整章节标题映射（含"第X章"），**下载初始化时即从网页预填所有章节**，用于EPUB生成准确目录 |
| `downloaded` | 已成功下载的章节索引列表 |
| `failed` | 下载失败的章节索引列表 |
| `status` | `in_progress` 进行中 / `complete` 已完成 |

> **注意：** 从 v1.3 开始，`chapter_titles` 在下载初始化时即从网页抓取所有章节并一次性写入，不再逐章追加。这意味着即使下载因故中断，`chapter_titles` 也保证完整。

## 十一、常见问题

### Q1: 下载中途中断了怎么办？
直接重新运行相同的命令，脚本会从上次中断处自动继续。

### Q2: 有章节下载失败了怎么办？
运行完成后查看提示的失败索引。使用 `--rd` 选项重新下载失败章节：
```powershell
py -3 download_novel.py --rd 32157
```

### Q3: 重试后仍然失败怎么办？
个别章节可能因服务器问题持续无法访问。可稍后再试，或手动检查该章节在浏览器中能否打开。如果网站正常运行而脚本仍然失败，可以调整 config.json 中的重试参数（增加 `max_attempts` 或 `backoff_base`）。

### Q4: 生成EPUB时报错"未找到章节目录"
请先运行 `download_novel.py` 完成下载，再运行 `build_epub.py`。

### Q5: 生成EPUB时中文显示乱码
EPUB 文件本身使用 UTF-8 编码，大多数现代阅读器（如 Calibre、Kobo、Kindle）都支持。如果遇到乱码，请检查阅读器的编码设置。

### Q6: 如何更改下载延迟？
编辑 `config.json` 中的 `request_delay.min` 和 `request_delay.max` 为你想要的秒数。

### Q7: 如何用代理运行脚本？
```powershell
# 设置环境变量
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
```

### Q8: 如何删除下载进度重新下载？
删除对应小说目录下的 `progress.json` 文件和 `chapters/` 文件夹，重新运行下载命令。

## 十二、文件说明

|------|------|
| `download_novel.py` | 章节下载脚本（支持 `--rd` 重试失败章节、`--check` 文件状态校验） |

| `<书名>/<书名>.epub` | 生成的EPUB电子书 |

## 十三、完整示例

以下是一次完整的下载+生成流程：

```powershell
# 1. 安装依赖（仅首次需要）
cd C:\Users\automation\Vibe_Coding\fetech_xs
py -3 -m pip install -r requirements.txt

# 2. 下载所有章节（预计耗时 8~10 小时）
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py 32157

# 3. （可选）如果怀疑状态不一致，先检查修复
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py --check 32157

# 4. （可选）如果某些章节失败，重试
$env:PYTHONIOENCODING='utf-8'; py -3 download_novel.py --rd 32157

# 5. 生成 EPUB 电子书
$env:PYTHONIOENCODING='utf-8'; py -3 build_epub.py 召唤圣剑

# 6. 找到生成的 EPUB 文件
#    在 fetech_xs/召唤圣剑/召唤圣剑.epub
```
