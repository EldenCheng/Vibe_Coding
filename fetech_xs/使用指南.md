# ftech_xs 使用指南

## 一、简介

ftech_xs 是一个小说下载与 EPUB 生成工具，专门用于从爱丽丝书屋 (ALICESW.COM) 下载小说章节并合成为 EPUB 电子书。

**核心功能：**
- 自动解析章节列表页面，提取所有章节链接
- 逐章下载章节内容（HTML 格式，保留段落结构）
- 内置反爬应对策略（延迟15~30秒、Cookie维持、空内容自动重试+API切换）
- 支持断点续传（中断后重新运行会自动继续）
- 下载完成后一键生成带目录的 EPUB 文件

## 二、环境准备

### 2.1 安装 Python

确保系统已安装 Python 3.13+。验证方式：

```powershell
py -3 --version
```

如果显示 `Python 3.13.5` 或类似版本，说明环境就绪。
如果提示找不到 `py`，请先安装 Python：https://www.python.org/downloads/

### 2.2 安装依赖

在 `ftech_xs/` 目录下运行：

```powershell
py -3 -m pip install -r requirements.txt
```

该命令会自动安装以下依赖：
| 包名 | 用途 |
|------|------|
| requests | HTTP请求与Session管理 |
| beautifulsoup4 | HTML解析 |
| lxml | 解析引擎加速 |
| ebooklib | EPUB电子书生成 |

## 三、快速开始

### 3.1 下载小说

```powershell
# 进入脚本目录
cd ftech_xs

# 方式一：用 book_id 下载（推荐）
py -3 download_novel.py 32157

# 方式二：用完整URL下载（适用于其他小说）
py -3 download_novel.py "https://xn--1jqx9a.als1004.space/other/chapters/id/32157.html"
```

**book_id 如何获取？**
打开你想下载的小说的"章节目录"页面，URL 末尾的数字就是 book_id。例如：
```
.../other/chapters/id/32157.html   ← 32157 就是 book_id
```

### 3.2 生成 EPUB

下载完成后，运行：

```powershell
py -3 build_epub.py 召唤圣剑
```

其中"召唤圣剑"是脚本自动创建的小说目录名（与小说网页上的书名一致）。
生成的文件在 `ftech_xs/召唤圣剑/召唤圣剑.epub`。

## 四、下载流程详解

### 4.1 第一步：解析章节列表

脚本访问章节列表页面，自动提取：
- **书名**：从 `<h1>《书名》</h1>` 中提取
- **作者**：从 `作者：<a>` 中提取
- **章节列表**：从 `<ul class="mulu_list">` 中提取所有含"章"的链接

### 4.2 第二步：逐章下载

```
每章循环：
  1. 随机等待 15~30 秒（模拟人工阅读，减轻服务器压力）
  2. 优先访问 HTML 页面下载内容
  3. 如果HTML返回空内容 → 自动切换到 AJAX API 接口重试
  4. 如果仍失败 → 按指数退避等待（30s, 60s, 120s, 240s, 480s）后重试
  5. 成功后保存为 <序号>_<标题>.html
  6. 立即更新 progress.json（断点续传依据）
```

### 4.3 第三步：生成 EPUB

脚本读取 `chapters/` 目录下的所有 HTML 文件，按序号排序后：
1. 创建 EpubBook 对象，设置书名、作者、语言
2. 为每章生成 EpubHtml（含章节标题样式 + 内容）
3. 自动生成目录（兼容 EPUB2 和 EPUB3 阅读器）
4. 输出 `.epub` 文件

## 五、断点续传

任何时候中断下载（按 Ctrl+C 或关机），重新运行相同的命令即可自动继续：

```powershell
# 重新运行，会从上次中断处继续
py -3 download_novel.py 32157
```

**原理：** 每成功下载一章后，都写入 `进度.json` 记录已下载的章节索引。下次启动时跳过这些章节。

## 六、下载多本小说

多次运行脚本即可，每本小说的章节会独立存放在各自的书名目录下：

```
ftech_xs/
├── 召唤圣剑/         ← 第一本书
│   ├── chapters/
│   └── progress.json
├── 另一本小说/       ← 第二本书
│   ├── chapters/
│   └── progress.json
└── download_novel.py
```

```powershell
py -3 download_novel.py 32157        # 下载《召唤圣剑》
py -3 download_novel.py <另一本书的ID>  # 下载另一本书
py -3 build_epub.py 召唤圣剑           # 生成第一本的EPUB
py -3 build_epub.py 另一本小说          # 生成第二本的EPUB
```

## 七、配置文件说明

`config.json` 位于 `ftech_xs/` 根目录，可以通过修改它来调整脚本行为：

```json
{
    "request_delay": {
        "min": 15,    // 每章最小等待时间（秒）
        "max": 30     // 每章最大等待时间（秒）
    },
    "retry": {
        "max_attempts": 5,       // 单章最大重试次数
        "backoff_base": 30,       // 首次重试等待时间（秒）
        "backoff_multiplier": 2   // 每次重试乘数（30→60→120→240→480）
    },
    "headers": {
        "User-Agent": "...",      // 浏览器标识
        "Accept": "...",          // HTTP请求头
        "Accept-Language": "zh-CN,zh;q=0.9",
        "Accept-Encoding": "gzip, deflate",
        "Connection": "keep-alive",
        "Cache-Control": "no-cache"
    }
}
```

**重要提醒：** 请合理设置 `request_delay` 的值，不要设置过小，避免给服务器造成过大负担。默认的15~30秒是安全合理的间隔。

## 八、常见问题

### Q1: 下载中途中断了怎么办？
直接重新运行相同的命令，脚本会从上次中断处自动继续。

### Q2: 某些章节下载失败怎么办？
失败的章节索引会记录在 `进度.json` 的 `failed` 数组中。目前脚本会自动跳过失败章节。如果希望重试失败章节，可以手动删除 `进度.json` 中 `failed` 数组里的索引后重新运行，或者删除 `进度.json` 重新下载全部。

### Q3: 生成EPUB时报错"未找到章节目录"
请先运行 `download_novel.py` 完成下载，再运行 `build_epub.py`。

### Q4: 生成EPUB时中文显示乱码
EPUB 文件本身使用 UTF-8 编码，大多数现代阅读器（如 Calibre、Kobo、Kindle）都支持。如果遇到乱码，请检查阅读器的编码设置。

### Q5: 如何更改下载延迟？
编辑 `config.json` 中的 `request_delay.min` 和 `request_delay.max` 为你想要的秒数。

### Q6: 如何用代理运行脚本？
```powershell
# 设置环境变量
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
```

# 2. 下载所有章节（预计耗时 8~10 小时）
