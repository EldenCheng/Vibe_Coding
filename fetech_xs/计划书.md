# 计划书 - ftech_xs 小说下载与EPUB生成工具

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 项目名称 | ftech_xs - 爱丽丝书屋小说下载与EPUB生成工具 |
| 目标网站 | `https://xn--1jqx9a.als1004.space` |
| 运行环境 | Python 3.13.5, 命令: `py -3` |
| 包管理 | `py -3 -m pip install -r requirements.txt` |

**设计原则**: 只需提供一个章节列表URL，脚本自动完成一切——提取书名、作者、章节链接、下载、生成EPUB。

## 二、技术栈

| 库 | 用途 |
|----|------|
| `requests` (已安装 2.34.0) | HTTP Session 管理、请求、重试 |
| `beautifulsoup4` | HTML 解析（章节列表、章节内容） |
| `lxml` | HTML/XML 解析后端加速 |
| `ebooklib` | EPUB 构建、目录生成 |
| `json`, `pathlib`, `re`, `time`, `random`, `sys` | 标准库 |

## 三、目录结构

```
ftech_xs/
├── 计划书.md
├── 版本变更.md
├── requirements.txt
├── config.json                    # 全局配置（延迟、重试、请求头）
├── download_novel.py              # 章节下载脚本
├── build_epub.py                  # EPUB生成脚本
└── <小说书名>/                    # 自动根据书名创建
    ├── progress.json              # 下载进度（断点续传）
    ├── chapters/                  # 章节HTML文件
    │   ├── 0001_新的旅程.html
    │   └── ...
    └── <小说书名>.epub            # 最终EPUB
```

## 四、配置文件设计

### 4.1 全局配置 `ftech_xs/config.json`

```json
{
    "request_delay": {
        "min": 15,
        "max": 30
    },
    "retry": {
        "max_attempts": 5,
        "backoff_base": 30,
        "backoff_multiplier": 2
    },
    "headers": {
        "User-Agent": "Mozilla/5.0 ... Chrome/125.0.0.0 Safari/537.36",
        "Accept": "text/html,...",
        "Accept-Language": "zh-CN,zh;q=0.9,en;q=0.8",
        "Accept-Encoding": "gzip, deflate",
        "Connection": "keep-alive",
        "Cache-Control": "no-cache"
    }
}
```

**不需要** 每本书单独建 config.json。小说名、作者、ID 等信息全部从章节列表页面自动提取。

## 五、模块设计

### 5.1 `download_novel.py` — 章节下载器

#### 用法

```powershell
# 方式一：传入完整URL
py -3 download_novel.py "https://xn--1jqx9a.als1004.space/other/chapters/id/32157.html"

# 方式二：仅传入ID（自动拼接URL）
py -3 download_novel.py 32157

# 方式三：重试失败章节（--rd）
py -3 download_novel.py --rd 32157
```

`--rd` 标志：下载状态为 complete 后，仅重试 `progress.json` 中 `failed` 列表内的章节。

#### 5.1.1 主流程

```
parse_args() → (retry_failed, [chapters_url])
  - 解析 --rd/-rd 标志和 positional URL 参数

main(chapters_url, retry_failed):
  1. 加载全局 config.json
  2. 创建 requests.Session()，设置 headers
  3. fetch_book_info(session, chapters_url)
     → {book_name, author, book_id, source_id, chapters[]}
  4. 创建 <book_name>/ 和 <book_name>/chapters/ 目录
  5. 加载/创建 <book_name>/progress.json

  情况A — retry_failed=True 且 status=complete:
    6a. 读取 progress.downloaded 和 progress.failed
    6b. 仅重试 failed 中的章节（按原始索引顺序）
    6c. 成功 → 从 failed 中移除
    6d. 仍失败 → 重新加入 failed（或保持原位）
    6e. 更新 status（failed 清空则 complete，否则 in_progress）

  情况B — 正常下载:
    6b. for each chapter not in progress.downloaded:
          sleep(random(15, 30))
          download_with_retry(session, chapter)
          save_chapter_to_file(chapter)
          写入 chapter_titles（完整标题含"第X章"）
          update_progress()
    7. 打印统计: 成功 X/总数, 失败 Y
```

#### 5.1.2 书籍信息提取 (`fetch_book_info`)

```
输入: session, chapters_url
输出: book_info 字典

从章节列表页提取:
  - 书名: <div class="mu_h1"><h1> 的文本内容（去掉"《》"）
  - 作者: <span>作者：<a> 的文本
  - book_id: 从 chapters_url 正则提取 /id/(\d+)
  - source_id: 从章节链接正则提取 /book/(\d+)/
  - chapters: 从 <ul class="mulu_list"> 所有 <li><a> 提取
    → 每项: {index, title, url, hash}
    → 过滤卷标链接（如"第一部 黄昏篇"）：标题不含"第.*章"且长度很短
```

#### 5.1.3 下载章节 (`download_chapter`)

```
输入: session, base_url, source_id, chapter_hash, chapter_index
输出: (success, html_content)

方式A — HTML页面下载:
  url = f"{base_url}/book/{source_id}/{chapter_hash}.html"
  设置 Referer = 章节列表页URL
  GET → 200 → 解析 <div class="read-content j_readContent user_ad_content">
  提取该 div 的 inner HTML（所有<p>标签）

方式B — AJAX API 备选（方式A返回空时自动切换）:
  url = f"{base_url}/home/chapter/info"
  params = {id: source_id, key: chapter_hash}
  设置 X-Requested-With: XMLHttpRequest
  GET → JSON → resp['data']['chapter']['content']

判定空内容:
  inner_html 去除空白字符后长度为 0 → 视为空内容
```

#### 5.1.4 空内容重试

```
attempt ∈ [1, max_attempts]:
  success, content = download_chapter(method="html")
  if not success:
    success, content = download_chapter(method="ajax")
  if success:
    保存文件, 更新进度 → 成功返回
  wait = backoff_base × (backoff_multiplier ^ (attempt-1)) 秒
  等待后重试

仍未成功 → 标记 failed, 记录日志
```

#### 5.1.5 章节文件保存

```
文件名: {index:04d}_{stripped_title}.html
  示例: 0001_新的旅程.html
  标题清理: 去掉第X章前缀, 去掉非法文件名字符 (? * : " < > |)

文件内容: 直接写入从页面提取的 inner HTML（保留 <p> 标签结构）
```

#### 5.1.6 断点续传

```
progress.json 格式:
{
    "book_name": "召唤圣剑",
    "author": "西贝猫",
    "chapters_url": "https://...",
    "total": 679,
    "downloaded": [0, 1, 2],
    "failed": [],
    "last_index": 2,
    "status": "in_progress",
    "chapter_titles": {
        "0": "第一章 新的旅程",
        "1": "第二章 圣剑套牌"
    }
}
每次下载完一个章节立即更新文件。
再次运行脚本时，跳过 downloaded 中的章节。
chapter_titles: 完整标题映射（含"第X章"），用于 build_epub.py 生成准确目录。
```

### 5.2 `build_epub.py` — EPUB生成器

#### 用法

```powershell
py -3 build_epub.py 召唤圣剑
```

#### 5.2.1 主流程

```
main(novel_dir):
  1. 检查 novel_dir/chapters/ 存在
  2. 从 progress.json 读取 author
  3. 扫描 chapters/ 目录下所有 .html 文件，按文件名数字前缀排序
  4. 从 chapter_titles 字段读取完整标题（优先于文件名推断）
  5. 读取每个 HTML 文件内容
  6. 构建 EpubBook:
     a. 设置元数据 (书名、作者、语言=zh-CN)
     b. 添加默认 CSS
     c. 为每个章节创建 EpubHtml（标题含完整"第X章"）
     d. 构建目录 (TOC) — 显示完整章节号+标题
     e. 添加 EpubNcx + EpubNav（兼容 EPUB2/EPUB3）
  7. epub.write_epub() → <novel_dir>/<书名>.epub
  8. 打印成功信息
```

## 六、运行流程总览

```
用户                              脚本
 │                                 │
 ├─ py -3 download_novel.py URL ──┤
 │                                 ├─ 加载 config.json
 │                                 ├─ GET 章节列表页 → 解析书名/作者/章节
 │                                 ├─ 创建目录结构
 │                                 ├─ 逐章下载 (延迟15-30s) ──→ chapters/*.html
 │                                 │   ├─ 章节标题(含"第X章") → progress.json chapter_titles
 │                                 │   └─ 失败章节 → progress.json failed 列表
 │                                 └─ 完成 ──→ "成功 679/679"
 │                                 │
 ├─ py -3 build_epub.py 书名 ─────┤
 │                                 ├─ 扫描 chapters/*.html
 │                                 ├─ 读取 chapter_titles → 完整标题
 │                                 ├─ 构建 EPUB（元数据+TOC(含章节号)+内容）
 │                                 └─ 完成 ──→ 书名.epub
 │                                 │
 │  (可选) 中途失败章节重试
 ├─ py -3 download_novel.py --rd ID ─┤
 │                                 ├─ 读取 failed 列表
 │                                 ├─ 仅下载失败章节
 │                                 └─ 成功 → 移除失败标记
```

## 七、错误处理

| 错误 | 处理 |
|------|------|
| config.json 缺失 | 使用内置默认值 |
| 章节列表页 GET 失败 | 重试3次，仍失败则退出 |
| 书名提取失败 | 用 URL 中的 ID 作为备选书名 |
| 章节 HTML 下载返回空 | 切换到 AJAX API → 重试 → 失败标记 |
| 网络超时 (30s) | 计入 attempt，退避重试 |
| HTTP 非200 | 同上 |
| chapters 目录下无文件 | build_epub 报错退出 |
| EPUB 写入失败 | 报错退出 |
| `--rd` 但 status=complete 且 failed 为空 | 提示无需重试，退出 |
| `--rd` 但 progress.json 不存在 | 提示先下载，退出 |
| chapter_titles 缺失 | build_epub 回退到文件名推断（无"第X章"前缀） |
