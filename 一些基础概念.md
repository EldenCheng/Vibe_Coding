# 什么是Agent, Skill与MCP

## Agent

现在的主流大模型都是对话模型, 大模型本来对用户正在使用的环境是没有任何感知力的, 也没有办法操作用户端的任何东西, 所以, 传统上如果要让它帮忙解决一些复杂的问题, 可能需要把一些基础的东西告诉大模型, 比如我要完成一个工作 - 我有一些数据分散在数个excel工作表上, 需要把这些数据集中在一起然后生成一个数据图. 那么我要使用大模型来帮助我工作, 我就必须要告诉大模型以下信息:

1. Excel的版本, 是xls还是xlsx

2. 各个Excel数据表的名字

3. 数据来源

4. 数据图的类型

5. 存放数据图的数据表

6. 使用的脚本语言, 比如我准备使用python来做这个任务

而大模型在收到我提供的讯息后, 经过分析, 会返回一系列建议给我, 可能会包括:

1. 需要用到的python语言的版本

2. python语言的依赖库

3. 完整的python语言逻辑

然后我就需要:

1. 按照建议, 检查python的版本号与依赖库, 如果不符合, 就需要我去安装对应的版本与依赖

2. 把大模型提供的逻辑复制粘贴到一个.py文件中

3. 使用命令行运行这个python逻辑

4. 检查最终结果

从上面的工作流程可以看到, 在这里大模型负责的就是对我提出的需求, 根据我拥有的环境提出建议, 让我再去执行. 中间很多的地方都需要人去参与的

但是, 一个简单的工作可以这样, 用户也不会学得很不方便, 如果这个工作变得很复杂呢? 又或者需要重复很多次呢? 怎么办, 还是大模型提出建议让用户人手去完成吗?
