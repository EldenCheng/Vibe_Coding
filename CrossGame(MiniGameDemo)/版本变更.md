#### 平行相邻约束算法

**问题**
- 原有的逐格子检查过于严格（禁止任何相邻），导致无法生成任何有效的PuzzleSet

**解决方案**
- 新算法：在所有单词放置完成后，检查任意两个**同方向**的单词是否存在格子相邻（共享边）
- 同方向单词相邻且不共享格子 → 视觉上误导用户（看起来像同一单词的一部分）
- 异方向单词交叉共享格子 → 允许（正常的填字游戏交叉）

**算法**
```js
// 收集 w1 的所有格子坐标
const w1Cells = new Set();
for (let i = 0; i < w1.word.length; i++) {
  const r = w1.direction === 'across' ? w1.row : w1.row + i;
  const c = w1.direction === 'across' ? w1.col + i : w1.col;
  w1Cells.add(`${r},${c}`);
}

// 检查 w2 的每个格子是否与 w1 的格子相邻
for (let i = 0; i < w2.word.length; i++) {
  const r = w2.direction === 'across' ? w2.row : w2.row + i;
  const c = w2.direction === 'across' ? w2.col + i : w2.col;
  const neighbors = [[r-1,c],[r+1,c],[r,c-1],[r,c+1]];
  for (const [nr, nc] of neighbors) {
    if (w1Cells.has(`${nr},${nc}`)) return true; // 违规
  }
}
```

## v0.6.0 — 2026-06-25

### 变更内容

#### 焦点导航优化（按单词顺序跳转）

**问题**
- 原有焦点导航按阅读顺序（从左到右、从上到下）跳转，导致竖向单词的字母顺序被打乱
- 填完一个单词后不会按单词编号顺序跳转到下一个单词

**解决方案**
- 焦点导航改为按单词顺序：填完当前单词的字母后，自动跳到下一个未完成单词的第一个空格
- 处理共享格：如果共享格已被前一个单词填入，后续单词导航时自动跳过
- 点击切换：用户点击某个格子时，在该格子所属的单词中找第一个未完成的，设为当前单词

**算法**
1. 构建导航数据结构：`#wordCellMap`（每个单词的格子列表）、`#cellToWords`（格子→单词映射）
2. 输入字母后，在当前单词中找下一个空格
3. 当前单词完成则跳到下一个未完成单词的第一个空格
4. 点击格子时切换当前单词（选第一个未完成的）

**示例**
```
单词1: CAT (横向) → (0,0)C (0,1)A (0,2)T
单词2: CAR (纵向) → (0,0)C (1,0)A (2,0)R

流程：
  填单词1: C→A→T  ✓
  切到单词2: 跳过已填的C → A→R  ✓
```
