## 1. 数据流概览

```
源词表 JSON          word-loader.js         puzzle-generator.js      data/*.json
(grade/terms/units)  → 过滤+去重合法单词  →  生成交叉布局PuzzleSet  →  游戏加载
      ↓                     ↓                        ↓                     ↓
  你的输入文件        自动处理，无需修改        自动处理，无需修改      需要手动注册
```

**关键文件说明：**

| 文件 | 作用 | 是否需要修改 |
|------|------|------------|
| `广州科教版小学英语三至五年级单词表.json` | 小学单词源文件 | 不需要（已有） |
| `scripts/word-loader.js` | 读取源词表，过滤合法单词 | 不需要 |
| `scripts/puzzle-generator.js` | 生成填字游戏棋盘布局 | 不需要 |
| `scripts/generate.js` | 入口脚本，协调生成流程 | 需要（修改配置） |
| `data/小学-填字游戏.json` | 生成的游戏数据 | 不需要（自动生成） |
| `data/data-sources.json` | 数据源注册表 | 需要（添加新条目） |

## 2. 准备单词库 JSON 文件

### 2.1 文件格式要求

单词库 JSON 文件必须遵循以下层级结构：

```json
[
  {
    "grade": "年级名称",
    "terms": [
      {
        "term": "学期名称",
        "units": [
          {
            "unit": "单元名称",
            "words": [
              {
                "en": "english_word",
                "zh": "中文释义"
              }
            ]
          }
        ]
      }
    ]
  }
]
```

### 2.2 字段说明

| 层级 | 字段 | 类型 | 说明 |
|------|------|------|------|
| grade | `grade` | string | 年级名称（如"七年级上册"） |
| grade | `terms` | array | 学期列表 |
| term | `term` | string | 学期名称 |
| term | `units` | array | 单元列表 |
| unit | `unit` | string | 单元名称（如"Unit 1 Getting to Know You"） |
| unit | `words` | array | 单词列表 |
| word | `en` | string | 英文单词（**必须**是纯英文字母） |
| word | `zh` | string | 中文释义 |

### 2.3 单词过滤规则

`word-loader.js` 会自动过滤不符合以下条件的单词：

- `en` 字段必须匹配正则 `/^[A-Za-z]{3,20}$/`
  - 纯英文字母（无数字、空格、连字符）
  - 长度 3-20 个字符
- `zh` 字段必须是非空字符串
- 同一单词库内自动去重（保留首次出现的条目）

**被过滤的常见原因：**
- 单词太短（如 "a", "I", "am"）
- 单词包含非字母字符（如 "don't", "ice-cream"）
- 单词太长（超过 20 个字母）

## 3. 生成 PuzzleSet 数据

### 3.1 修改 generate.js 配置

编辑 `scripts/generate.js`，修改以下配置项：

```javascript
// 修改前（小学配置）
const WORD_LIST_PATH = resolve(__dirname, '../广州科教版小学英语三至五年级单词表.json');
const OUTPUT_FILE = '小学-填字游戏.json';
const WORDS_PER_PUZZLE = 5;

// 修改后（中学配置）
const WORD_LIST_PATH = resolve(__dirname, '../中学英语七年级单词表.json');
const OUTPUT_FILE = '中学-填字游戏.json';
const WORDS_PER_PUZZLE = 4;  // 推荐 4 词/组
```

**配置项说明：**

| 配置项 | 说明 | 建议值 |
|--------|------|--------|
| `WORD_LIST_PATH` | 源词表 JSON 文件路径 | 指向你的新单词库文件 |
| `OUTPUT_FILE` | 输出的数据文件名 | 放在 `data/` 目录下 |
| `WORDS_PER_PUZZLE` | 每组 PuzzleSet 包含的单词数 | **4**（推荐） |
| `COVERAGE_TARGET` | 目标覆盖率（0-1） | 0.9（90%） |
| `MAX_CONSECUTIVE_FAILURES` | 最大连续失败次数 | 50000 |

### 3.2 运行生成脚本

在项目根目录执行：

```bash
node scripts/generate.js
```

**预期输出：**

```

1/3 加载单词表...
   ✅ 加载到 451 个合法单词

2/3 生成 PuzzleSet...

3/3 保存文件...
```

### 3.3 验证生成结果

生成完成后，检查 `data/中学-填字游戏.json` 文件：


## 4. 将新单词库添加到游戏

### 4.1 注册数据源

编辑 `data/data-sources.json`，添加新条目：

```json
[
  {
    "displayName": "小学",
    "dataPath": "data/小学-填字游戏.json"
  },
  {
    "displayName": "中学",
    "dataPath": "data/中学-填字游戏.json"
  }
]
```

**字段说明：**

| 字段 | 说明 | 示例 |
|------|------|------|
| `displayName` | 游戏中显示的名称 | "中学" |
| `dataPath` | 数据文件路径（相对于项目根目录） | "data/中学-填字游戏.json" |

### 4.2 验证添加成功

1. 启动本地服务器：
   ```bash
   start-server.bat
   ```
   或
   ```bash
   py -m http.server 8001
   ```

2. 打开浏览器访问 `http://localhost:8001/start.html`

3. 在首页的下拉菜单中应该能看到新增的"中学"选项

4. 选择"中学"并开始游戏，验证数据加载正常

## 6. 常见问题

### Q: 生成失败，提示"未能生成任何有效 PuzzleSet"

**可能原因：**
- 单词库中合法单词太少（少于 10 个）
- 单词之间缺少可交叉的字母组合

**解决方法：**
- 检查单词库是否包含足够多的 3-20 字母单词
- 增加 `MAX_CONSECUTIVE_FAILURES` 值（如 100000）
- 尝试减少 `WORDS_PER_PUZZLE`（如改为 3）

### Q: 生成的 PuzzleSet 数量太少

**可能原因：**
- 单词库中可交叉的单词组合有限

**解决方法：**
- 增加源词表中的单词数量
- 降低 `COVERAGE_TARGET`（如 0.7）
- 接受较少的 PuzzleSet 数量（游戏会循环使用）

### Q: 游戏中看不到新添加的单词库

**检查项：**
1. `data-sources.json` 格式是否正确（JSON 语法）
2. `dataPath` 路径是否正确（相对路径，从项目根目录）
3. 数据文件是否存在且内容有效
4. 刷新浏览器缓存

### Q: 如何为多个年级生成数据？

**方法一：合并为一个文件**
将多个年级的单词合并到一个 JSON 文件中，然后运行一次生成。

**方法二：分别生成**
分别为每个年级准备单词库文件，修改 `generate.js` 多次运行，每次输出不同文件名。

### Q: 如何修改每组单词数量？

编辑 `scripts/generate.js` 中的 `WORDS_PER_PUZZLE` 常量：

- **4 词/组**（推荐）：生成更快，棋盘更紧凑
- **5 词/组**：生成较慢，棋盘更大
- **3 词/组**：最容易生成，但每组单词较少
