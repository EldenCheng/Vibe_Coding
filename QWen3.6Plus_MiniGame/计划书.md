# 井字过三关 (Tic-Tac-Toe) 项目计划书

## 1. 项目概述

### 1.1 项目名称
QWen3.6_MiniGame - 井字过三关

### 1.2 项目描述
基于纯 HTML/CSS/JavaScript 实现的井字过三关游戏,支持双人对战与人机对战,采用星空科幻主题风格。

### 1.3 技术栈
- HTML5
- CSS3
- JavaScript (ES6+)
- Web Audio API (音效)

---

## 2. 功能需求

### 2.1 核心功能
| 功能 | 描述 |
|------|------|
| 棋盘 | 标准 3x3 棋盘 |
| 双人对战 | 两名玩家本地轮流下棋 |
| 人机对战 | 玩家 vs AI,可选难度 |
| 胜负判定 | 自动检测胜利/平局/继续 |

### 2.2 AI 难度
| 难度 | 策略 |
|------|------|
| 简单 | 随机落子 |
| 中等 | 50% 概率最优解,50% 随机 |
| 困难 | Minimax 算法,始终最优解 |

### 2.3 计分板
- 记录当前会话的胜/负/平次数
- 区分玩家X与玩家O的战绩
- 人机模式下记录玩家 vs AI 的战绩
- 关闭浏览器后清空(不使用 localStorage)

### 2.4 游戏历史记录
- 记录每局游戏的结果(胜/负/平)
- 显示对局序号、模式、结果
- 可滚动查看历史列表
- 关闭浏览器后清空(不使用 localStorage)

### 2.5 音效
| 音效类型 | 触发时机 |
|----------|----------|
| 落子音效 | 每次放置棋子时 |
| 胜利音效 | 玩家获胜时 |
| 平局音效 | 双方平局时 |
| 失败音效 | 玩家输掉时(人机模式) |

> 注: 使用简单的固定音效,不依赖外部音频文件,无音量调节功能

## 3. 技术架构

### 3.1 文件结构
```
GLMTest_MiniGame/
├── index.html          # 主页面
├── css/
│   └── style.css       # 样式表
├── js/
│   ├── game.js         # 游戏核心逻辑
│   ├── ai.js           # AI 对手逻辑
│   ├── ui.js           # UI 交互控制
│   ├── sound.js        # 音效管理
│   └── history.js      # 历史记录管理
├── assets/
│   └── (预留,可能存放图标等)
├── 计划书.md           # 本文件
└── 版本变更.md          # 版本变更记录
```

### 3.2 模块职责

#### game.js - 游戏核心逻辑
- 棋盘状态管理
- 落子逻辑
- 胜负判定算法
- 游戏流程控制(开始/重置/结束)

#### ai.js - AI 对手
- 简单模式: 随机选择空位
- 中等模式: 混合策略
- 困难模式: Minimax 算法

#### ui.js - UI 交互
- 模式选择(双人/人机)
- 难度选择
- 棋盘渲染与点击事件
- 计分板更新
- 游戏状态提示

#### sound.js - 音效管理
- Web Audio API 封装
- 音效播放接口

#### history.js - 历史记录
- 历史数据存储(内存)
- 历史记录渲染

## 4. UI 设计

### 4.1 视觉风格 - 星空主题
- **背景**: 深色渐变 + 星星动画(使用 CSS 动画或 Canvas)
- **棋盘**: 半透明发光边框,星空质感
- **棋子 X/O**: 霓虹发光效果,不同颜色区分
- **按钮**: 半透明 + 悬停发光效果
- **字体**: 科技感无衬线字体

### 4.2 配色方案
| 元素 | 颜色 |
|------|------|
| 背景 | #0a0a2e → #1a1a4e (深蓝渐变) |
| 星星 | #ffffff (闪烁) |
| 棋盘边框 | #4a9eff (发光蓝) |
| 棋子 X | #ff6b6b (珊瑚红) |
| 棋子 O | #4ecdc4 (青绿) |
| 胜利高亮 | #ffd93d (金色) |
| 文字 | #e0e0ff |

├─────────────────────────────────┤
│         棋盘区域                 │
│      ┌───┬───┬───┐              │
│      │   │   │   │              │

│      │   │   │   │              │
│      ├───┼───┼───┤              │
│      │   │   │   │              │

### 4.4 响应式
- 不需要响应式适配,仅针对桌面端浏览器优化
