# 项目设计文档：自动化测试用例生成工具

## 1. 项目概述

### 1.1 项目目标

编写一个Python脚本，实现从输入文件（PDF/图片）到生成测试矩阵（Excel）的完整自动化流程。

### 1.2 核心流程

```
输入文件 → 文件类型检测 → 处理为图片 → API逐个分析生成markdown → API逐个生成test case JSON → API生成Excel test matrix
```

### 1.3 运行方式

```bash
py main.py --input <输入文件路径> [--prompts <可选提示词1,可选提示词2>] [--temp-prompt "临时提示词内容"]
```

## 2. 项目结构

```
│   ├── 基础生成提示词.md        # 生成test case的基础提示词（预留接口）
│   ├── 基础格式提示词.md        # 生成Excel格式的提示词
│   └── Flowchart分析提示词.md   # Flowchart专用分析提示词
├── main.py                      # 主入口脚本
├── config_manager.py            # 配置管理模块
├── prompt_manager.py            # 提示词管理模块
├── file_processor.py            # 文件类型检测和处理模块
├── pdf_converter.py             # PDF转图片模块
├── api_client.py                # API调用封装模块
├── analysis_processor.py        # 分析处理模块（图片→markdown，独立可运行）
├── test_case_generator.py       # 测试用例生成模块（独立可运行）
├── test_matrix_generator.py     # 测试矩阵生成模块（markdown→Excel，独立可运行）
├── 项目提示词.md                 # 原始需求文档
├── 项目设计文档.md               # 本文档
└── 版本变更.md                   # 版本变更记录
```

## 3. 输出结构

当用户指定输入文件后，所有输出文件存放在**输入文件所在目录**下的`outputs`子目录中：

```
输入文件所在目录/
├── outputs/
│   ├── images/                    # PDF转换的图片（page_1.jpg, page_2.jpg, ...）
│   ├── analysis/                  # 分析结果（page_1.md, page_2.md, ...）
│   ├── test_cases.json            # 测试用例JSON（所有页面的测试用例合并）
│   └── test_matrix_v1.xlsx        # 测试矩阵（带版本编号）
└── Flowchart.pdf                  # 输入文件
```

### 3.1 版本编号规则

- 第一次生成的测试矩阵编号为 `v1`
- 之后每次生成自动递增：`v2`, `v3`, ...
- 通过检测outputs目录中已有的`test_matrix_v*.xlsx`文件，确定下一个版本号

## 4. 命令行接口

### 4.1 参数说明

| 参数              | 短参数  | 说明              | 必需  | 默认值         |
| --------------- | ---- | --------------- | --- | ----------- |
| `--input`       | `-i` | 输入文件路径（PDF或图片）  | 是   | 无           |
| `--prompts`     | `-p` | 可选提示词组合，多个用逗号分隔 | 否   | 无（只使用基础提示词） |
| `--temp-prompt` |      | 临时提示词内容         | 否   | 无           |
| `--help`        | `-h` | 打印帮助信息          | 否   |             |

### 4.2 使用示例

```bash
# 基本用法：只使用基础分析提示词
py main.py --input Flowchart.pdf

# 使用可选提示词
py main.py --input Flowchart.pdf --prompts Flowchart

# 使用多个可选提示词
py main.py --input Flowchart.pdf --prompts Flowchart,AnotherPrompt

# 使用临时提示词
py main.py --input Flowchart.pdf --temp-prompt "请特别关注状态机的转换逻辑"

# 完整用法
py main.py --input Flowchart.pdf --prompts Flowchart --temp-prompt "特殊分析要求"

# 查看帮助
py main.py -h
```

### 4.3 帮助信息格式

`-h` 参数的帮助信息需要动态读取`config/prompts_mapping.json`，列出所有可用的可选提示词名称。示例：

```
用法: py main.py [选项]

自动化测试用例生成工具

选项:
  -i, --input FILE        输入文件路径（PDF或图片）[必需]
  -p, --prompts PROMPTS   可选提示词组合，多个用逗号分隔 [可选]
                          可用的可选提示词: Flowchart
  --temp-prompt TEXT      临时提示词内容 [可选]
  -h, --help              显示此帮助信息
```

### 4.4 参数验证

- `--input`：文件必须存在，且扩展名必须是`.pdf`、`.jpg`、`.jpeg`、`.png`、`.bmp`、`.gif`
- `--prompts`：每个提示词名称必须在`config/prompts_mapping.json`中存在
- 如果参数验证失败，输出错误信息并退出

## 5. 模块设计

### 5.1 config_manager.py - 配置管理模块

**职责**：管理所有配置文件的读取和验证

**类**：`ConfigManager`

**方法**：

- `__init__(self, project_root)` - 初始化，project_root为项目根目录路径
- `load_auth_config(self)` → dict - 读取`config/auth.json`，返回`{"base_url": "...", "api_key": "..."}`
- `load_prompts_mapping(self)` → dict - 读取`config/prompts_mapping.json`，返回提示词映射字典
- `get_available_prompts(self)` → list - 获取所有可用的可选提示词名称列表
- `validate_prompt_name(self, name)` → bool - 验证提示词名称是否有效

**配置文件格式**：

`config/auth.json`：

```json
{
  "base_url": "http://192.168.1.3:8080/v1",
  "api_key": "EMPTY"
}
```

`config/prompts_mapping.json`：

```json
{
  "Flowchart": "prompts/Flowchart分析提示词.md"
}
```

### 5.2 prompt_manager.py - 提示词管理模块

**职责**：管理提示词的读取和拼接

**类**：`PromptManager`

**方法**：

- `__init__(self, project_root)` - 初始化
- `get_base_analysis_prompt(self)` → str - 读取基础分析提示词内容
- `get_base_generation_prompt(self)` → str - 读取基础生成提示词内容
- `get_base_format_prompt(self)` → str - 读取基础格式提示词内容
- `get_optional_prompt(self, name)` → str - 根据名称读取可选提示词内容
- `build_analysis_prompt(self, optional_prompts=None, temp_prompt=None)` → str - 拼接分析提示词组合
  - 拼接顺序：基础分析提示词 → 可选提示词（按用户指定顺序）→ 临时提示词
- `build_generation_prompt(self)` → str - 拼接生成test case提示词（目前只返回基础生成提示词）
- `build_format_prompt(self)` → str - 拼接格式提示词（目前只返回基础格式提示词）

**提示词拼接规则**：

1. 基础分析提示词始终作为第一部分
2. 可选提示词（用户选择）作为第二部分，按用户指定的顺序拼接
3. 临时提示词（用户输入）作为最后一部分
4. 各部分之间用换行符分隔

### 5.3 file_processor.py - 文件类型处理模块

**职责**：检测文件类型，分发到对应处理函数

**函数**：

- `detect_file_type(file_path)` → str - 检测文件类型，返回"pdf"/"image"/"unsupported"
- `process_file(file_path, output_dir)` → bool - 处理文件，返回是否成功
  - PDF → 调用`pdf_converter.convert_pdf_to_images()`
  - 图片 → 复制到`output_dir/images/`目录
  - 不支持的格式 → 输出错误信息

**支持的文件扩展名**：

- PDF: `.pdf`
- 图片: `.jpg`, `.jpeg`, `.png`, `.bmp`, `.gif`
- 不支持: `.docx`, `.xlsx`, `.pptx`等

### 5.4 pdf_converter.py - PDF转图片模块

**职责**：将PDF的每一页转换为图片

**依赖**：PyMuPDF (fitz)

**函数**：

- `convert_pdf_to_images(pdf_path, output_dir)` → bool - 将PDF每页转为图片，保存到output_dir

**参数**：

- `pdf_path`: PDF文件路径
- `output_dir`: 图片保存目录（如`outputs/images/`）

**输出格式**：JPG格式，文件名格式为`page_1.jpg`, `page_2.jpg`, ...

**图片质量**：使用`fitz.Matrix(2, 2)`进行2倍缩放，确保清晰度

### 5.5 api_client.py - API调用模块

**职责**：封装大模型API调用，支持图片和文本两种模式

**类**：`APIClient`

**方法**：

- `__init__(self, base_url, api_key, model="gemma-4-local")` - 初始化API客户端
- `analyze_image(self, image_path, prompt_text)` → str - 分析图片内容（多模态模式）
  - 将图片转为Base64编码
  - 使用OpenAI兼容的多模态API格式
  - 返回大模型的回答文本
- `analyze_text(self, prompt_text)` → str - 分析文本内容（纯文本模式）
  - 使用OpenAI兼容的文本API格式
  - 返回大模型的回答文本

**API调用格式**（图片模式）：

```python
{
    "role": "user",
    "content": [
        {"type": "text", "text": prompt_text},
        {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{base64_image}"}}
    ]
}
```

**API调用格式**（文本模式）：

```python
{
    "role": "user",
    "content": prompt_text
}
```

### 5.6 analysis_processor.py - 分析处理模块

**职责**：逐个读取图片，调API生成分析结果（markdown），支持独立运行和被main.py调用两种方式

**类**：`AnalysisProcessor`

**方法**：

- `__init__(self, api_client, prompt_manager)` - 初始化
- `process_images(self, images_dir, output_dir, optional_prompts=None, temp_prompt=None)` → bool
  - 逐个读取`images_dir`中的图片文件（按文件名排序）
  - 对每个图片调用`api_client.analyze_image()`
  - 将结果保存为markdown文件到`output_dir`（如`analysis/page_1.md`）
  - 返回是否全部成功

**独立运行方式**：

```bash
py analysis_processor.py --images-dir output/images --output-dir output/analysis [--prompts Flowchart] [--temp-prompt "特殊分析要求"]
```

**独立运行参数**：

| 参数               | 说明              | 必需           |
| ---------------- | --------------- | ------------ |
| `--images-dir`   | 图片目录路径          | 是            |
| `--output-dir`   | 分析结果输出目录路径      | 是            |
| `--prompts`      | 可选提示词组合，多个用逗号分隔 | 否            |
| `--temp-prompt`  | 临时提示词内容         | 否            |
| `--project-root` | 项目根目录路径         | 否（默认为脚本所在目录） |

**处理流程**：

1. 获取`images_dir`中所有图片文件（按文件名排序）
2. 对每个图片：
   - 构建提示词组合（使用`prompt_manager.build_analysis_prompt()`）
   - 调用`api_client.analyze_image()`获取分析结果
   - 将结果保存为对应的markdown文件
3. 输出处理进度

### 5.7 test_case_generator.py - 测试用例生成模块

**职责**：从分析结果生成test case JSON，支持独立运行和被main.py调用两种方式

**类**：`TestCaseGenerator`

**方法**：

- `__init__(self, api_client, prompt_manager)` - 初始化
- `generate_test_cases(self, analysis_dir, output_json_path)` → bool
  - 逐个读取`analysis_dir`中的markdown文件
  - 使用基础生成提示词 + 分析结果拼接后调API
  - 将结果合并保存到`output_json_path`（支持追加模式）
- `_clean_json_response(self, raw_content)` → str - 清理API返回的JSON，处理LLM常见的格式问题：
  - 移除Markdown代码块标记（```json...```）
  - 提取第一个`{`到最后一个`}`之间的内容（处理前后多余文本）
  - 移除JSON中的注释（`//`和`/* */`）
  - 处理单引号问题（将单引号替换为双引号）
  - 移除尾部逗号（在`}`或`]`前面的逗号）
  - 移除控制字符
- `_load_existing_json(self, json_path)` → dict - 加载已有的JSON文件
- `_merge_test_cases(self, existing_data, new_data, page_name)` → dict - 合并多个页面的test case

**独立运行方式**：

```bash
py test_case_generator.py --analysis-dir output/analysis --output output/test_cases.json
```

**独立运行参数**：

| 参数               | 说明          | 必需           |
| ---------------- | ----------- | ------------ |
| `--analysis-dir` | 分析结果目录路径    | 是            |
| `--output`       | 输出的JSON文件路径 | 是            |
| `--project-root` | 项目根目录路径     | 否（默认为脚本所在目录） |

**输出JSON格式**（参考Round3）：

```json
{
  "summary": {
    "modes": [
      {
        "mode_name": "Mode Name",
        "total": 0,
        "executed": 0,
        "passed": 0,
        "failed": 0,
        "skipped": 0
      }
    ]
  },
  "modes_data": {
    "Mode Name A": [
      {
        "test_id": 1,
        "test_description": "...",
        "expected_result": "...",
        "results": "",
        "comment": "",
        "internal_bug_id": "",
        "external_bug_id": ""
      }
    ]
  }
}
```

### 5.8 test_matrix_generator.py - 测试矩阵生成模块

**职责**：读取test case JSON，本地生成Excel格式的test matrix（使用openpyxl），支持独立运行和被main.py调用两种方式

**类**：`TestMatrixGenerator`

**方法**：

- `__init__(self)` - 初始化（无需API客户端）
- `generate_excel(self, test_cases_json_path, output_dir)` → str - 生成Excel文件
  - 读取test case JSON文件
  - 使用openpyxl本地生成Excel（不调用API）
  - 自动检测版本号（v1, v2, ...）
  - 保存到`output_dir/test_matrix_v{n}.xlsx`
  - 返回生成的文件路径
- `_sanitize_sheet_name(self, name)` → str - 清理sheet名称（移除Excel不允许的字符: `[]:*?/\`，截断到31字符）
- `_create_summary_sheet(self, wb, modes_data)` - 创建Summary sheet（含COUNTIF公式，使用清理后的sheet名称）
- `_create_mode_sheet(self, wb, mode_name, test_cases)` - 创建Mode sheet（含颜色编码）
- `_classify_test_area(test_case)` - 根据测试用例内容判断测试区域

**测试区域颜色编码**：

- 绿色 `#C6EFCE`：Happy Path / Functional
- 红色 `#FFC7CE`：Negative Input（带[TBD]标记）
- 黄色 `#FFEB9C`：Boundary & Counter
- 蓝色 `#BDD7EE`：Timeouts & States
- 紫色 `#D9D2E9`：Interrupts & Exceptions

**独立运行方式**：

```bash
py test_matrix_generator.py --input-json output/test_cases.json --output-dir output
```

**独立运行参数**：

| 参数             | 说明                 | 必需  |
| -------------- | ------------------ | --- |
| `--input-json` | test case JSON文件路径 | 是   |
| `--output-dir` | 输出目录路径             | 是   |

**版本号检测逻辑**：

1. 扫描`output_dir`中已有的`test_matrix_v*.xlsx`文件
2. 提取所有已有版本号，找到最大值
3. 新版本号 = 最大值 + 1
4. 如果没有已有文件，版本号为1

## 6. 主入口 main.py

### 6.1 职责

- 解析命令行参数
- 初始化所有模块
- 协调整个处理流程

### 6.2 处理流程

```
1. 解析命令行参数
2. 验证输入文件
3. 确定输出目录（输入文件所在目录/output/）
4. 初始化配置管理器，加载配置
5. 初始化提示词管理器
6. 初始化API客户端
7. 处理文件：
   a. 检测文件类型
   b. PDF → 转换为图片 → 保存到output/images/
   c. 图片 → 复制到output/images/
8. 分析处理：
   a. 逐个读取output/images/中的图片
   b. 调API生成分析结果（markdown）
   c. 保存到output/analysis/
9. 生成test case：
   a. 逐个读取output/analysis/中的markdown
   b. 调API生成test case JSON
   c. 合并保存到output/test_cases.json
10. 生成test matrix：
    a. 读取test case JSON
    b. 调API生成Excel
    c. 保存到output/test_matrix_v{n}.xlsx
11. 输出完成信息
```

### 6.3 错误处理

- 输入文件不存在：输出错误信息并退出
- 无效的文件类型：输出错误信息并退出
- 无效的可选提示词名称：输出错误信息并退出
- API调用失败：输出警告信息，继续处理下一个文件
- 配置文件读取失败：输出错误信息并退出
