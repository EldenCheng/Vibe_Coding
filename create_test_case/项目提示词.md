## 总体设计

1. 编写一个python脚本, 读取我指定的一个文件后
   
   1. 分析文件的类型
      
      1. 如果是pdf, 调用下面定义的另一个pdf处理python脚本处理好pdf
      
      2. 如果是图片, 直接保存在我指定文件同目录下的outputs子目录中, 指定文件夹的默认值可以是当前路径
      
      3. 如果是word, excel, ppt, 暂时先不处理, 输出信息告诉用户暂不支持
   
   2. PDF处理脚本
      
      1. 把pdf文件的每一页转换成图片, 然后把转换好的图片放到指定文件同目录下的outputs子目录中

2. 在处理好指定的文件后, 参考"api_image_demo.py"与"api_file_demo.py", 编写一个python脚本用于把上面处理好的输出文件通过API发送到大模型, 让它分析, 理解图片, 然后把它分析, 理解的结果转换成structured logic文档
   
   1. 为了安全起见, 大模型API的BaseURL与API key不能直接写在脚本中, 要另外从"auth.json"中读取(从api_demo中读取相关内容, 然后建议这个文件, 再在python脚本中动态读取)
   
   2. 脚本应该自动判断读取的处理结果文件是图片或者md文件, 如果是图片, 就使用传图片的API结构, 如果是md文件, 就使用传md文件内容的API结构
   
   3. python脚本应该允许用户指定提示词组合, 然后根据用户指定的提示词组合动态拼接生成发给大模型API的最终提示词
      
      1. 所有提示词在prompts子目录中
      
      2. 基础提示词是"基础分析提示词.md"
         
         1. 如果用户没有显性指定提示词, 默认值就是使用"基础分析提示词.md"作为发给API的提示词
         
         2. 如果用户显性指定了提示词组合(可以多个), "基础分析提示词.md"也应该作为提示词的第一部分, 然后与其它提示词拼接后, 作为给API的提示词. 另外, 应该在config子目录下维护一个可选提示词名称与实际文件路径的组合的mapping Json, 比如"Flowchart": "/prompts/Flowchart分析提示词.md", 意思是, 如果用户指定的提示词组合中包含了Flowchart, 脚本应该从mapping Json里知道找哪一个文件是对应的为Flowchart而设的提示词
         
         3. 应该允许用户输入的临时提示词, 临时提示词的默认值应该是没有, 但如果用户有输入, 则应该作为提示词组合的最后一部分, 拼接到前面拼接好的提示词组合的最后, 然后作为给API的提示词
   
   4. 因为大模型上下文长度有限, 所以处理输出文件要一个一个读取, 处理. 处理完一个, 生成了一个结果后, 再处理下一个
   
   5. 把处理后的结果放到输入的文件相同的目录中(也就是上面提到的我指定文件同目录下的outputs子目录)

3. 在生成了所有structured logic文档, 编写一个python脚本, 从指定路径(也就是上面提到的我指定文件同目录下的outputs子目录)读取生成的structured logic, 配合生成提示词, 通过API发送到大模型, 让它按顺序生成test case, 然后把生成test case按顺序放到同一个Json文件里
   
   1. 所有提示词在prompts子目录中
   
   2. 如果用户没有显性指定提示词, 默认值就是使用"基础生成提示词.md"作为发给API的提示词
   
   3. 
   
   4. 使用Json格式生成test case数据, 然后把数据按顺序存放到同一个Json文件里. 后面生成的test case追加到Json文件的后面, 最后会得到一个包括所有test case的Json文件
   
   5. 因为大模型上下文长度有限, 所以处理输出文件要一个一个读取, 处理. 处理完一个, 生成了一个结果后, 再处理下一个

4. 最后参考"api_file_demo.py", 生成一个python脚本, 根据包含test case数据的完整Json文件与提示词, 生成excel 格式的test matirx
   
   1. 所有提示词在prompts子目录中
   
   2. 使用"基础格式提示词.md"作为生成的提示词
   
   3. 生成的test matrix要加上编号, 第一次生成的编号为v1, 之后如此类推
   
   4. 生成的结果存放到输入的文件相同的目录中(也就是上面提到的我指定文件同目录下的outputs子目录)
