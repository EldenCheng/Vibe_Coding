# Flowchart Master Protocol (综合流程图分析总纲)

## 第一阶段：页面识别与分类 (Page Classifier)

在分析任何图像前，请首先判断该页面的类型。根据视觉特征将页面归类为以下三种模式之一。**必须在分析开头明确写出： “页面识别结果：[模式 A / 模式 B / 模式 C]”**。

### 模式 A：状态机风格 (State Machine / Interaction Flow)

**视觉特征：**

* **主状态**：大量使用**圆角矩形**（如 "Happy Mode", "Neutral Mode"）。
* **全局中断**：出现圆形或小圆点 $\rightarrow$ 矩形 的结构，表示高优先级抢占。
* **全局规则**：常伴有黄色填充矩形定义不可中断规则。
* **逻辑核心**：[当前状态] $\xrightarrow{事件/触发}$ [动作] $\rightarrow$ [新状态]。

### 模式 B：判定树/链路风格 (SPM / Decision Tree / Linear Flow)

**视觉特征：**

* **生命周期**：起点和终点使用椭圆形（如 "Transformation Flow", "Go to ... Mode"）。
* **连续判定**：主干由一系列连续的菱形（Yes/No 分支）组成。
* **循环结构**：出现回溯箭头，指向之前的动作矩形以形成循环。
* **跨块跳转**：一页中存在多个独立流程块，通过 "Go to [X] Flow" 实现跳转。
* **逻辑核心**：[起点] $\rightarrow$ [条件判定] $\rightarrow$ [同步动作] $\rightarrow$ [终点]。

### 模式 C：文档/概述页 (Documentation / Overview Page)

**视觉特征：**

* **无流程元素**：没有典型的菱形、圆角矩形或逻辑连接箭头。
* **文本密集**：包含大标题、版本历史记录 (Version History)、产品描述 (Product Description) 或硬件规格 (Hardware Specs)。
* **呈现形式**：以列表、段落或简单的静态文本表形式呈现。
* **逻辑核心**：提供项目的背景上下文、版本状态及硬件约束。

## 第二阶段：执行分析协议 (Execution Protocol)

### 如果判定为【模式 A】，请执行 $\text{CreativityFlowchart分析提示词.md}$：

1. **识别状态清单**（圆角矩形）。
2. **分析主路径**（状态 $\rightarrow$ 触发 $\rightarrow$ 动作）。
3. **识别中断机制**（圆形节点 $\rightarrow$ 响应）。
4. **检查全局约束**（黄色矩形）。
5. **资源映射分析**。

### 如果判定为【模式 B】，请执行 $\text{SPMFlowchart分析提示词.md}$：

1. **确定生命周期**（椭圆起点 $\rightarrow$ 椭圆终点）。
2. **追踪判定链路**（菱形 Yes/No 分支）。
3. **分析循环与同步动作**（回溯箭头、同矩形多文本）。
4. **构建跨流程拓扑**（识别跳转关系与全局中断块）。
5. **资源映射分析**。

### 如果判定为【模式 C】，请执行【背景知识提取协议】：

1. **版本识别**：提取当前文档的版本号（如 v10b）及其变更重点。
2. **硬件清单**：详细列出所有 Inputs（传感器/按钮）和 Outputs（LED/扬声器）的硬件规格。
3. **功能概述**：总结产品的核心行为逻辑和产品定位。
4. **上下文存储**：将上述信息记录为“背景知识”，用于辅助后续 A/B 模式的流程分析（例如：将流程图中的 "Head pat" 关联到硬件页的 "Capacitive head button"）。

## 第三阶段：通用资源提取 (Universal Asset Extraction)

对于模式 A 和 模式 B，统一提取以下资源标识：

* $\color{blue}{\text{蓝色字体}}$ $\rightarrow$ **LED 资源**
* $\color{red}{\text{红色字体}}$ $\rightarrow$ **GFX (图形) 资源**
* $\color{green}{\text{绿色字体}}$ $\rightarrow$ **Sound (声音) 资源**
* **表格内容** $\rightarrow$ 记录资源映射、参数定义或动态难度曲线。

## 输出要求

1. **第一行**：必须明确写出 **“页面识别结果：[模式 A / 模式 B / 模式 C]”**。
2. **后续内容**：严格按照对应模式的执行协议输出分析结果。
