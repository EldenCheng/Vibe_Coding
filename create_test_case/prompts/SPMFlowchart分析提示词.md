# SPMFlowchart 分析提示词

## 1. 视觉元素定义表 (Legend)
在分析此类流程图时，请严格遵守以下形状与逻辑映射关系：

*   **生命周期节点 (Lifecycle Nodes)**：**椭圆形** $\rightarrow$ 表示该流程的边界。
    *   **起点 (Entry Point)**：位于流程最顶端或最左侧的椭圆（如 "Transformation Flow"）。
    *   **终点 (Exit Point)**：位于流程末端，指示跳转至其他模式或流程的椭圆（如 "Go to Sleep Mode", "Go to Blooming flow"）。
*   **判断逻辑 (Decision)**：**菱形** $\rightarrow$ 所有菱形内部均视为一个判断条件。必须追踪其发出的 **"yes"** 和 **"no"** 分支路径。
*   **执行动作 (Action)**：**矩形** $\rightarrow$ 表示系统执行的具体指令或物理动作。
    *   **同步动作 (Simultaneous Actions)**：若同一个矩形内包含多个描述（如 "PCLOSE" 与 "Close petals"），表示这些动作是**同时发生**或**等效执行**的。
*   **循环与回溯 (Loop-back)**：**回溯箭头** $\rightarrow$ 当判定结果为 `no` 或不满足跳出条件时，箭头回指至之前的动作矩形，表示该动作将**反复执行**直到条件满足。
*   **资源标识 (Asset Designators)**：沿用颜色标识 $\rightarrow$ **蓝色=LED**, **红色=GFX(图形)**, **绿色=Sound(声音)**。

## 2. 跨流程关联逻辑 (Cross-Flow Logic)
一张图纸可能包含多个独立的流程块，它们之间存在以下两种关系：
*   **模块化跳转 (Modular Jump)**：当一个流程的终点椭圆标注为 `Go to [X] Flow` 时，它实际上是一个跳转指令，指向另一个流程块的起点。
*   **全局中断/监听 (Global Interrupt)**：某些独立流程块在后台并行运行。一旦其中的判定条件满足，将立即**中断**当前正在执行的其他任何流程，并抢占执行该中断流程的动作。

## 3. 分析步骤 (Analysis Workflow)
请按以下顺序分析，确保逻辑链路完整：

**Step 1: 识别流程分片 (Fragmentation Analysis)**
*   扫描整页，识别出所有独立的流程块。
*   为每个流程块命名（根据其起点椭圆）。

**Step 2: 追踪内部判定与循环**
*   从每个块的起点开始，沿着箭头追踪。
*   分析所有**菱形**的 `yes/no` 分支。
*   识别**回溯箭头**，明确循环执行的动作及其跳出条件。

**Step 3: 建立跨块关联**
*   记录所有 `Go to ...` 的终点 $\rightarrow$ 寻找对应的起点，构建整体跳转拓扑图。
*   识别没有明显入口或由全局状态触发的流程块，将其标记为“全局中断流程”。

**Step 4: 识别动作与资源**
*   列出执行动作，特别标注同步动作。
*   提取所有彩色字体，归类到 LED、GFX 或 Sound 资源清单。

## 4. 输出要求 (Output Format)
请以以下格式汇报分析结果：
1.  **流程拓扑**：列出所有流程块及其跳转关系（例如：[流程A] $\xrightarrow{终点}$ [流程B起点]）。
2.  **逻辑判定树**：为每个流程块详细列出判定条件、分支结果及循环逻辑。
3.  **动作序列**：详细列出执行的动作，并注明同步发生的动作。
4.  **中断机制**：列出所有被识别为“全局中断”的流程块及其触发条件。
5.  **资源清单**：按颜色分类列出所有涉及的底层资源 ID。
